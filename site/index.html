The audit trail for AI-written code.

Everything AI touched in your repos: ordered, traceable, in one place. Flagger records who wrote each commit, how confident the attribution is, and how much human review it received.

Connect GitHub How attribution works

One install. Whole-org visibility.

Competing tools ask every developer to wrap their Git workflow in a CLI. Flagger connects at the repository level, once, and starts recording immediately.

  1. Install the GitHub App

    Pick the organizations and repositories you want on the record. No per-developer tooling, workflow changes, or agent SDKs.

  2. Activity flows in over webhooks

    Pushes, pull requests, reviews, and CI runs are ingested as they happen, so every record reflects the process around the change.

  3. Read the ledger

    One feed shows what agents wrote, the diff, the review it got, and the signals that make a change worth a second look.

Certain or suspected. Never blurred.

Attribution is a claim about who wrote your code, so Flagger tells you which kind of claim it is making on every commit.

certain

Read directly from evidence such as git-ai notes at refs/notes/ai, agent and model included, or commits pushed by a known agent bot account.

suspected

Inferred from heuristics: Co-authored-by trailers naming a known agent, commit-message patterns, and authorship signals. A heuristic match is never dressed up as a fact.

Already using git-ai? Flagger reads its data format natively as a complementary source rather than a competitor.

What Flagger is not.

An audit trail is only as good as its honesty, including about itself.

  • Not a code reviewer

    Flagger records provenance and process. It does not judge whether code is good.

  • Not a scoreboard

    No streaks, leaderboards, or per-developer adoption scores. Flagger measures activity, not people.

  • Not an alarm wall

    Risk signals are simple and transparent: no review, unclean CI, sensitive paths, large unreviewed diffs, and direct pushes to main.

  • Not a data silo

    Every row links back to the commit, diff, pull request, and review on GitHub.
