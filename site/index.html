One install. Whole-org visibility.

Competing tools ask every developer to wrap their Git workflow in a CLI. Flagger connects at the repository level, once, and starts recording immediately.

  1. Install the GitHub App

    Pick the organizations and repositories you want on the record. That's the whole setup: no per-developer tooling, no workflow changes, no agent SDKs.

  2. Activity flows in over webhooks

    Pushes, pull requests, reviews, and CI runs are ingested as they happen. Every commit is attributed at ingestion and re-evaluated as review and CI evidence arrives, so a commit's record reflects what actually happened to it.

  3. Read the ledger

    One feed across every repo and contributor: what agents wrote, the diff, the review it got, and the signals that make a change worth a second look, like unreviewed pushes to main, large diffs nobody read, and sensitive paths.

Certain or suspected. Never blurred.

Attribution is a claim about who wrote your code, so Flagger tells you which kind of claim it's making, on every single commit.

certain

Read directly from the evidence. If a repo writes git-ai notes at refs/notes/ai, Flagger uses them as its highest-confidence source, agent and model included. Commits pushed by a known agent's own bot account (devin-ai-integration[bot]) are attributed the same way.

suspected

Inferred from heuristics: Co-authored-by trailers naming a known agent, commit-message patterns, authorship signals. Useful, and labeled as exactly what it is. A heuristic match is never dressed up as a fact.

Already using git-ai? Flagger reads its data format natively, as a complementary source rather than a competitor. Where it isn't present, heuristics fill in, honestly labeled.

What Flagger is not.

An audit trail is only as good as its honesty, including about itself.

  • Not a code reviewer

    Flagger doesn't judge whether code is good; that's your review process's job. It records where code came from and how much human oversight it got. Provenance and process, not quality scores.

  • Not a scoreboard

    No streaks, no leaderboards, no per-developer adoption scores. The moment AI usage becomes a metric, it becomes a game. Flagger measures activity, not people.

  • Not an alarm wall

    Risk signals in v1 are simple, transparent flags: no review, unclean CI, sensitive paths, large unreviewed diffs, direct pushes to main. You can see exactly why each one fired. No opaque scores, no unvalidated ML dressed as fact.

  • Not a data silo

    Every row links back to the commit, diff, PR, and review on GitHub. The ledger survives app removal, because an audit trail that disappears when you uninstall isn't an audit trail.
